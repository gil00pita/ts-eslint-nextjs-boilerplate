# Illustrations

Illustrations are used sparingly in Phenix UI. They should support comprehension, not decorate an interface that already has enough information.

## When To Use

| Scenario      | Guidance                                                           |
| ------------- | ------------------------------------------------------------------ |
| Empty states  | Show what kind of content belongs here or what happened.           |
| Onboarding    | Help orient people to a new workflow.                              |
| Major success | Use only when the moment deserves more warmth than a status badge. |
| Error states  | Avoid whimsy; focus on recovery and next steps.                    |

## Visual Direction

- Simple geometric forms with clear silhouettes.
- Limited color, usually neutral surfaces with one Ember or Sky accent.
- Human figures only when they clarify scale or collaboration.
- No decorative background blobs or abstract atmosphere.

## Production Requirements

- Provide descriptive alt text when the image adds meaning.
- Use empty alt text when the image is decorative.
- Keep file sizes small and export at the required display size.
- Test illustrations in light and dark themes.
