# Themes

Phenix UI supports light, dark, and high-contrast themes through semantic tokens. Components should read tokens, not inspect color mode directly.

## Theme Modes

<table>
  <thead>
    <tr>
      <th>Theme</th>
      <th>Use</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>Light</td>
      <td>Default for most business workflows and documentation.</td>
    </tr>
    <tr>
      <td>Dark</td>
      <td>Useful for monitoring, long-running operations, and low-light environments.</td>
    </tr>
    <tr>
      <td>High contrast</td>
      <td>Required for users who need stronger text, border, and focus contrast.</td>
    </tr>
  </tbody>
</table>

  <thead>

      </td>
    </tr>

    </tr>
  </tbody>
</table>

## Rules

- Prefer semantic tokens in component usage.
- Define light and dark values at the token layer.
- Test focus, hover, selected, invalid, and disabled states in every theme.
- Avoid mode-specific ternaries in application components.
