# Typography

Phenix UI typography is designed for scanning, decision-making, and sustained reading. It uses a restrained scale with clear hierarchy.

## Type Roles

| Role          | Token        | Use                                      |
| ------------- | ------------ | ---------------------------------------- |
| Display       | `display.lg` | Rare product moments and overview pages. |
| Page title    | `heading.xl` | Main page heading.                       |
| Section title | `heading.md` | Major regions within a workflow.         |
| Body          | `body.md`    | Default reading text.                    |
| Compact       | `body.sm`    | Tables, dense forms, and metadata.       |
| Label         | `label.sm`   | Form labels and field-level UI.          |
| Code          | `code.sm`    | Technical values, ids, and snippets.     |

## Writing With Type

- Use one page title per page.
- Keep button labels short and verb-led.
- Use compact text only when density helps the task.
- Do not rely on size alone for hierarchy; pair type with structure and spacing.

## Line Length

| Content            | Recommended width                                |
| ------------------ | ------------------------------------------------ |
| Long-form guidance | 60 to 80 characters per line.                    |
| Forms              | Align labels and inputs to the surrounding grid. |
| Tables             | Prefer compact text with clear headers.          |
| Alerts             | Keep to one main message and one next step.      |
