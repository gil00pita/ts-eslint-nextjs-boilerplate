# Available Components

The component catalogue is grouped by product job rather than implementation detail.

## Inventory

| Group        | Components                                              | Status |
| ------------ | ------------------------------------------------------- | ------ |
| Actions      | Button, IconButton, ActionBar, SplitButton              | Stable |
| Forms        | Field, Input, Textarea, Select, Checkbox, Radio, Switch | Stable |
| Feedback     | Alert, Toast, EmptyState, Progress, Skeleton            | Stable |
| Navigation   | Breadcrumb, Tabs, SideNav, Pagination                   | Beta   |
| Data display | Table, DataCard, Stat, Timeline, StatusBadge            | Beta   |
| Overlays     | Dialog, Drawer, Popover, Tooltip, Menu                  | Stable |
| Layout       | Page, PageHeader, Stack, Grid, Container                | Stable |

## Maturity Levels

| Status     | Meaning                                                    |
| ---------- | ---------------------------------------------------------- |
| Draft      | API and design are still changing.                         |
| Beta       | Ready for product pilots, but feedback may change details. |
| Stable     | Safe for broad adoption with documented migration support. |
| Deprecated | Supported only while teams migrate to a replacement.       |

## Component Acceptance Criteria

- Uses Phenix semantic tokens and recipes.
- Supports keyboard interaction and visible focus states.
- Includes empty, loading, disabled, invalid, and overflow states where relevant.
- Has documentation for when to use it and when to avoid it.
