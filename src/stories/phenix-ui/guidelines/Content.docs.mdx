# Content

Phenix UI content should help people act with confidence. Use plain language, name the object or task clearly, and put the consequence close to the action.

## Voice

<table>
  <thead>
    <tr>
      <th>Quality</th>
      <th>Guidance</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>Direct</td>
      <td>Say what happened and what to do next.</td>
    </tr>
    <tr>
      <td>Calm</td>
      <td>Do not exaggerate urgency.</td>
    </tr>
    <tr>
      <td>Specific</td>
      <td>Name the item, status, or requirement when it helps recovery.</td>
    </tr>
    <tr>
      <td>Respectful</td>
      <td>Avoid blame and avoid making assumptions about user intent.</td>
    </tr>
  </tbody>
</table>

## Rules

- Start button labels with a verb when possible.
- Avoid "Are you sure?" without explaining the consequence.
- Write errors in complete, helpful sentences.
- Keep alert titles short and put detail in the body.
