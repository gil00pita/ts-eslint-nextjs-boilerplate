# Content

Phenix UI content should help people act with confidence. Use plain language, name the object or task clearly, and put the consequence close to the action.

## Voice

| Quality    | Guidance                                                      |
| ---------- | ------------------------------------------------------------- |
| Direct     | Say what happened and what to do next.                        |
| Calm       | Do not exaggerate urgency.                                    |
| Specific   | Name the item, status, or requirement when it helps recovery. |
| Respectful | Avoid blame and avoid making assumptions about user intent.   |

## Common Patterns

| Pattern            | Example                          |
| ------------------ | -------------------------------- |
| Primary action     | `Save changes`                   |
| Destructive action | `Delete workspace`               |
| Error summary      | `Fix 3 fields before continuing` |
| Empty state title  | `No pending reviews`             |
| Loading label      | `Loading account history`        |

## Rules

- Start button labels with a verb when possible.
- Avoid "Are you sure?" without explaining the consequence.
- Write errors in complete, helpful sentences.
- Keep alert titles short and put detail in the body.
