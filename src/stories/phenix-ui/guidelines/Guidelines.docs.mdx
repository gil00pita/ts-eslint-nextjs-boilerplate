# Guidelines

Phenix UI guidelines help teams make consistent product decisions after choosing a component. They cover accessibility, content, and repeated patterns.

## What Guidelines Are For

<table>
  <thead>
    <tr>
      <th>Guideline</th>
      <th>Helps teams decide</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>Accessibility</td>
      <td>
        Whether an experience works for people using different input methods, assistive technology,
        or display settings.
      </td>
    </tr>
    <tr>
      <td>Content</td>
      <td>What to say, how to say it, and when to be brief.</td>
    </tr>
    <tr>
      <td>Patterns</td>
      <td>Which reusable flow shape fits a task.</td>
    </tr>
  </tbody>

## Working Agreement

- Components provide a strong baseline, not automatic accessibility.
- Content is part of the interface and should be reviewed with the same care as layout.
- Patterns should be adapted to the user journey, not copied blindly.
- When guidance conflicts with user research, bring the evidence back to the system team.
