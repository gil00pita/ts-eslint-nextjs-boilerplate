# What's New

This fictional changelog shows how Phenix UI communicates recent system work to product teams.

## 0.9.0 - Component Readiness

Released July 2026.

| Area          | Change                                                                        |
| ------------- | ----------------------------------------------------------------------------- |
| Components    | Added `DataCard`, `StatusBadge`, `EmptyState`, `PageHeader`, and `FilterBar`. |
| Forms         | Standardized field help text, invalid states, and required markers.           |
| Documentation | Added usage guidance for dashboard, service portal, and triage workflows.     |
| Accessibility | Added focus-ring guidance and keyboard interaction checklists.                |

## 0.8.0 - Brand Foundations

Released June 2026.

- Added the ember, ash, sky, moss, and berry color palettes.
- Published typography roles for product pages, forms, tables, and editorial guidance.
- Added radius and shadow tokens for dense application surfaces.
- Introduced light, dark, and high-contrast themes.

## 0.7.0 - First Internal Pilot

Released May 2026.

- Started pilot adoption with support tooling and account management teams.
- Added initial component inventory and contribution workflow.
- Defined content principles for alerts, confirmations, and irreversible actions.

## Release Notes Format

Each release should include:

| Field           | Guidance                                                      |
| --------------- | ------------------------------------------------------------- |
| Version         | Use semantic versioning for package releases.                 |
| Date            | Use the calendar date the package became available.           |
| Highlights      | Summarize the most important product impact first.            |
| Migration notes | Call out breaking changes, renamed props, and removed tokens. |
