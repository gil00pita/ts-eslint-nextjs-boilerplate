# Phenix UI

Phenix UI is a fictional design system for resilient product teams building workflow-heavy digital services. It gives teams a shared language for interface decisions, reusable React components, brand foundations, and accessibility standards.

The system is designed for products that must feel calm under pressure: operations dashboards, service portals, support tooling, and user journeys where clarity matters more than decoration.

## Principles

| Principle            | What it means                                                                      |
| -------------------- | ---------------------------------------------------------------------------------- |
| Clear by default     | Screens should explain state, next steps, and consequences without extra training. |
| Built for recovery   | Empty, loading, error, and partial-success states are first-class design work.     |
| Efficient at scale   | Components support dense information, keyboard use, and repeated workflows.        |
| Brand with restraint | The Phenix identity adds warmth and recognition without overpowering content.      |

## Package Shape

```tsx
import { Button, Card, PhenixProvider } from 'phenix-ui'

export function App() {
  return (
    <PhenixProvider>
      <Card.Root>
        <Card.Body>
          <Button colorPalette="ember">Review case</Button>
        </Card.Body>
      </Card.Root>
    </PhenixProvider>
  )
}
```

Phenix UI follows Chakra UI v3 conventions: compound components, semantic tokens, `colorPalette`, and provider-based theming.
